# Unicode Glyph Map

Every user-facing string in veil-frame's UI is rendered through a custom
Unicode glyph substitution. This keeps the visual identity consistent across
the app — both in hardcoded HTML and in JS-generated text.

## The Map

Defined in `static/app.js` as `unicode_lower` and applied via `stylizeUi()`.

**Pattern:** vowels (a, e, i, o, u) use **Bold** variants; consonants use
**regular** Mathematical Sans-Serif. This creates the characteristic mixed-weight
look throughout the UI.

## How to Apply

### In JavaScript (dynamic text)
```js
const label = stylizeUi('choose photo');
// → '𝖼𝗁𝐨𝐨𝗌𝐞 𝗉𝗁𝐨𝗍𝐨'
```

### In HTML templates (hardcoded text)
Run the text through the map manually and paste the result. A quick helper:

```python
# scripts/stylize.py  (or run in a REPL)
MAP = {
    'a': '𝐚', 'b': '𝖻', 'c': '𝖼', 'd': '𝖽', 'e': '𝐞',
    'f': '𝖿', 'g': '𝗀', 'h': '𝗁', 'i': '𝐢', 'j': '𝗃',
    'k': '𝗄', 'l': '𝗅', 'm': '𝗆', 'n': '𝗇', 'o': '𝐨',
    'p': '𝗉', 'q': '𝗊', 'r': '𝗋', 's': '𝗌', 't': '𝗍',
    'u': '𝐮', 'v': '𝗏', 'w': '𝗐', 'x': '𝗑', 'y': '𝗒',
    'z': '𝗓',
}

def stylize(text: str) -> str:
    return ''.join(MAP.get(c, c) for c in text.lower())

# Usage:
print(stylize('choose photo'))
```

## Rules for AI Coding Agents

1. **All** user-visible text in HTML templates must use the glyph map.
   Do not write plain ASCII — it will look wrong.
2. Text generated by JS should go through `stylizeUi()` — this is already
   the convention in `static/app.js`.
3. Non-alphabetic characters (digits, punctuation, spaces, emoji) pass through
   unchanged.
4. The `<select>` option text and `placeholder` attributes also use the map.
5. `text-transform: lowercase` in CSS works on normal ASCII but does NOT
   affect these Unicode codepoints — they are already lowercase by design.
